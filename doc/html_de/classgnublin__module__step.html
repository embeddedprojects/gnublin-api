Ausführliche Beschreibung

Klasse für den zugriff auf das GNUBLIN module-step.

Das GNUBLIN module-step lässt sich mit Hilfe der gnublin_step API ganz einfach ansteuern. Das Modul nutzt die I2C-Schnittstelle.

Beschreibung der Konstruktoren und Destruktoren

gnublin_module_step::gnublin_module_step ( )

Setzt die Standartwerte für irun und vmax (irun = 15, vmax = 8).

int gnublin_module_step::burnNewAddress ( int  new_address)

Brennt die übergebene slave Adresse.

Diese Funktion berechnet die benötigten Bits, um die neue Adresse auf den Chip zu brennen. Anschließend werden die OTP bits gebrannt. Achtung: Sind die Bits einmal gesetzt, kann dies nicht wieder rückgängig gemacht werden. Falls eine Adresse eingegeben wird, die nicht möglich ist, wird dies von der Funktion gemeldet.

int gnublin_module_step::getMotionStatus ( )

Bewegungs-Status.

Diese Funktion gibt den aktuellen bewegungs Status des module-step zurück.
0: Aktuelle Position = Ziehl Position; Geschwindigkeit= 0
1: Positive Beschleunigung; Geschwindigkeit > 0
2: Negative Beschleunigung; Geschwindigkeit > 0
3: Beschleunigung = 0; Geschwindigkeit = maximum pos Geschwindigkeit
4: Aktuelle Position /= Ziehl Position; Geschwindigkeit= 0
5: Positive Beschleunigung; Geschwindigkeit < 0
6: Positive Beschleunigung; Geschwindigkeit < 0
7: Acceleration = 0 Geschwindigkeit = maximum neg Geschwindigkeit
-1: Fehler

Rückgabe
motionStatus
